# BeDAO 管理后台系统

## Overview
BeDAO 管理后台是一个专业的加密货币交易平台后台管理系统，基于 Next.js 15 和 React 构建。该系统专注于为平台运营人员提供全面的管理功能，包括用户管理、交易监控、财务管理、系统配置等。本项目是纯后台管理系统，不包含任何用户端功能，旨在提供一个全面的、高效的、安全的运营管理平台，确保加密货币交易平台的顺畅运行和资产安全。

## User Preferences
Preferred communication style: Simple, everyday Chinese language.

## System Architecture

### Frontend
- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS, shadcn/ui, custom design system
- **State Management**: React Context API
- **Charting**: Recharts
- **Icons**: Lucide React
- **Internationalization**: Chinese/English language toggle

### Backend
- **Server**: Express.js with session management
- **Database**: PostgreSQL with Drizzle ORM
- **Authentication**: Replit Auth (OpenID Connect)
- **Session Storage**: PostgreSQL-backed session store

### Core Features
- **管理后台系统**: 综合性后台管理面板，包含安全的登录认证机制。
  - **Authentication**: 独立的管理员登录界面。
  - **新导航架构**: 采用顶部+左侧双层导航体系，一级菜单在顶部，二级菜单在左侧。
  - **15个管理模块**: 涵盖运营报表、用户管理、IM管理、社交管理、法币管理、担保管理、U卡管理、现货管理、合约管理、跟单管理、理财管理、佣金管理、BePay管理、财务管理、系统管理。
  - **访问控制**: 登录后才能访问管理功能，支持邮箱验证码。
  - **会话管理**: 持久的管理员认证状态。
- **BePay管理模块**: 包含9个子页面
  - **币种管理**: 管理支持的法币币种
  - **供应商管理**: 管理支付供应商
  - **接口管理**: 管理支付接口
  - **通道管理**: 管理支付通道
  - **商户管理**: 管理商户信息
  - **代理商管理**: 管理代理商及佣金比例
  - **代收订单**: 管理代收订单，包含补发/校验/冻结/退款功能
  - **代付订单**: 展示代付订单，支持币种/通道/状态筛选
  - **兑换订单**: 展示货币兑换订单，支持源/目标币种及状态筛选
- **UI/UX Decisions**:
  - **全新导航架构**: 顶部导航栏显示15个一级模块菜单，左侧边栏动态显示当前选中模块的二级子菜单。
  - **响应式导航系统**: 桌面端（≥768px）为横向滚动菜单，移动端（<768px）为汉堡菜单侧边Sheet弹窗。
  - 动态亮色/暗色主题切换，持久化用户偏好。
  - 统一的品牌设计（BeDAO）和标准化配色方案（绿色、黑色、深蓝色、白色）。
  - 登录页面独立布局。
  - **页面优化（BePay管理模块）**: 
    - 表格布局优化：通道管理(10→7列)、商户管理(9→6列)、代理商管理(9→5列)、订单管理(12→7列)，采用垂直堆叠设计合并相关信息
    - 删除顶部统计卡片：移除4个页面（通道/商户/代理商/订单管理）的顶部KPI卡片，让页面更简洁专注于核心功能
    - 统一页面结构：移除4个页面的外层白色背景容器，标题、筛选、搜索直接展示，仅表格部分使用白色卡片，与其他页面风格保持一致
    - 统一间距规范：所有BePay模块页面外层容器统一使用 `p-6 space-y-6`（24px padding + 24px垂直间距），确保与币种管理、接口管理等页面的视觉一致性
  - **代理商管理功能**：
    - 将原"佣金管理"重构为"代理商管理"，提供更全面的代理商管理功能
    - 页面说明：给自己配置了佣金大于0的用户则自动成为支付代理商，未配置过佣金的用户则不显示在本页面
    - 展示代理商基本信息（代理商ID、代理商名称、商户ID、UserID、邮箱、电话、佣金数据、商户数量、启用/冻结开关）
    - 代理商本身即是商户，也是用户，因此同时显示商户ID和UserID
    - **启用/冻结开关**：使用开关控制代理商状态，取代原有的状态徽章
      - 开关打开（绿色）：已启用状态
      - 开关关闭（灰色）：已冻结状态
      - 关闭开关时弹出冻结对话框，要求选择币种（CNY/USD/USDT/EUR/GBP）和输入冻结金额
      - 打开开关时直接解冻，状态变为"已启用"
      - 开关旁显示当前状态文字（"已启用"/"已冻结"）
    - "佣金比例"功能：点击查看该代理商在各支付通道的佣金比例设置（通道ID、通道名称、支付方式、佣金比例%、更新时间）
    - "详情"功能：查看代理商的佣金详细记录（记录ID、订单号、商户ID、订单金额、佣金比例、佣金金额、创建时间、状态）
      - **分页功能**：每页显示50条记录，支持翻页浏览
      - **日期筛选**：支持按开始日期和结束日期筛选订单
      - **商户ID搜索**：支持按商户ID搜索订单
      - **分页控件**：显示当前页、总页数、记录数统计，支持上一页/下一页导航
    - 支持按代理商名称、邮箱、电话、代理商ID、商户ID、UserID进行搜索过滤
    - **排名筛选系统**：5个排名页签（今日佣金排名、昨日佣金排名、本月佣金排名、历史佣金排名、商户数排名）
      - 前三名显示奖牌图标（🥇🥈🥉），第4名及之后显示数字排名
      - 根据选中的排名类型，佣金数据列动态显示对应的佣金金额（不同颜色区分）
      - 支持实时排序，自动按选中维度从高到低排列
  - **商户管理功能**：
    - 展示商户基本信息（商户名称、商户ID、UserID、邮箱、电话、账户余额、状态）
    - 商户本身也是用户，因此同时显示商户ID和UserID
    - **密钥管理**：删除成功率列，改为密钥列，支持查看商户的多个API密钥
      - 每个商户可以有多个API密钥
      - 点击"查看密钥"按钮显示密钥列表对话框
      - 每个密钥对应一个回调主域名
      - 显示密钥ID、API密钥、回调主域名、创建时间、域名状态
      - **域名审核功能**：
        - 每个API密钥有域名状态（已审核/待审核）
        - 已审核域名显示绿色"已审核"徽章
        - 待审核域名显示橙色"待审核"徽章，并提供审核操作按钮
        - "审核通过"按钮：将域名状态改为已审核，自动清除商户的待审核标记
        - "拒绝"按钮：删除该API密钥（需要确认操作）
        - 状态同步：审核操作实时更新商户列表和对话框状态
    - **新域名待审核提示**：独立于商户状态的域名审核标识
      - 域名由商户自己提交，需要后台审核
      - 显示在"查看密钥"按钮下方
      - 橙色标签显示"新域名待审核"提示
      - 商户状态包括：正常、冻结、禁用
    - **费率配置系统**：按币种和支付通道精细化配置费率，支持百分比费率和固定最低费用
      - **币种页签切换**：使用Tabs组件按币种（CNY、USD、USDT、EUR、GBP）分页签展示费率配置
      - **Inline实时编辑**：直接在表格中修改费率，无需点击编辑按钮
      - 表格列：支付通道、代收费率、最低代收费、代付费率、最低代付费、操作
      - **费率结构**：每个币种/通道组合包含4个费率字段
        - 代收费率（绿色）：百分比费率，如"0.5%"
        - 最低代收费（绿色）：固定最低手续费，如"¥1.00"
        - 代付费率（橙色）：百分比费率，如"0.3%"
        - 最低代付费（橙色）：固定最低手续费，如"¥0.50"
      - **添加费率配置**：支持添加新的币种/通道费率组合，立即显示在对应币种页签中
      - **删除费率配置**：删除后立即从表格中移除，自动切换到剩余币种页签
      - **状态同步**：添加、删除、编辑操作实时更新对话框状态和页签列表
      - 费率说明提示框，清晰解释百分比费率和最低手续费的作用
    - 支持按商户名称、邮箱、商户ID、UserID进行搜索过滤
  - **订单管理页面**：
    - **代收订单**：原"订单管理"重命名，包含完整管理功能
      - 收款信息：紫色Info图标按钮，点击查看收款账户详情（支持支付宝/微信/银行转账/PIX/UPI等多种支付方式）
      - 补发通知：只有成功订单可用，其他状态按钮灰色禁用
      - 重新校验：只有失败订单可用，其他状态按钮灰色禁用
      - 冻结订单：只有成功订单可用，其他状态按钮灰色禁用
      - 订单退款：只有成功订单可用，其他状态按钮灰色禁用
      - 禁用状态视觉反馈：50%透明度 + 禁止光标样式
      - 状态筛选：下拉框形式（全部/待支付/成功/失败），与搜索框在同一行
    - **代付订单**：包含完整管理功能，与代收订单样式完全一致
      - 付款信息：紫色Info图标按钮，点击查看付款账户详情（根据支付通道显示不同的付款信息）
      - 补发通知、重新校验、冻结订单、订单退款功能
      - 币种/支付通道筛选，支持订单搜索
      - 按钮状态控制逻辑与代收订单相同
      - 状态筛选：下拉框形式（全部/待支付/成功/失败），与搜索框在同一行
    - **兑换订单**：完整的三级页签筛选系统和汇率确认管理
      - 一级页签：划出账户（法币资产、法币代付金、现货USDT、U卡USDT）
      - 二级页签：划入账户（法币资产、法币代付金、现货USDT、U卡USDT）
      - 用户行为分类：下拉框形式（全部/法币换U/代付金划转/快捷充U），与状态筛选和搜索框在同一行
      - 状态筛选：下拉框形式（全部/待确认汇率/已完成/已拒绝），与用户行为分类和搜索框在同一行
      - 汇率确认功能：待确认汇率状态显示为黄色按钮，可点击输入两个汇率（用户汇率、成本汇率）
      - 自动利润计算：系统自动计算并显示USDT利润（成本USDT - 用户USDT）
      - 实时利润显示：输入汇率后即时显示用户获得USDT、成本USDT和利润
      - 汇率显示：待确认订单汇率显示为红色"待确认"字样
- **System Design Choices**:
  - 客户端路由系统：使用 `instant-navigation` 组件处理所有页面导航，避免页面重载。
  - 自定义路由机制：使用 `window.history.pushState` 和 `popstate` 事件实现即时页面切换。
  - **页面切换加载动画**: 切换左侧菜单时显示旋转加载器和"加载中..."文字，提供视觉反馈（300ms 过渡）。
  - 状态管理：使用 React Context API（ThemeProvider、AdminProvider）管理全局状态。
  - 安全的会话处理和路由保护（未登录时自动跳转登录页）。
  - 自动扩展部署配置，支持 Next.js 服务器功能和独立输出。
  - **双层导航架构**: 采用顶部一级菜单 + 左侧二级菜单的设计。
